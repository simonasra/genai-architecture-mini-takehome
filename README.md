# LLM System Design Screening Task

A 20-30 minute written design critique exercise for assessing LLM system design literacy before architecture interviews.

---

## Purpose

This repository contains a flawed design for an Analytics-Aware Chatbot. Your task is to review the design, identify critical risks, and propose mitigations. This exercise assesses:

- Security and privacy awareness
- LLM system design fundamentals
- Ability to prioritize and communicate clearly

No coding is required. Your deliverable is a single markdown file.

---

## For Candidates

### Instructions

1. Read the context — Start with [`docs/CONTEXT.md`](docs/CONTEXT.md) to understand the company, roles, and constraints.
2. Review the flawed design — Study [`docs/DESIGN.md`](docs/DESIGN.md) carefully. The design has intentional issues.
3. Complete your response — Fill in [`CANDIDATE_RESPONSE.md`](CANDIDATE_RESPONSE.md) following the template.
4. Submit — Commit your completed `CANDIDATE_RESPONSE.md` or share as instructed.

### Time Limits

| Limit | Duration |
|-------|----------|
| Recommended | 20 minutes |
| Hard stop | 30 minutes |

### Rules

- You may use LLMs, search engines, or any reference materials.
- The response must reflect your own thinking and judgment.
- Keep your response to one page maximum (~500 words).
- Do not modify any files except `CANDIDATE_RESPONSE.md`.
